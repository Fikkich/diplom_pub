# Структура проекту

Проект розділено на модулі для кращої організації коду:

## Модулі

### `config.py`
**Призначення:** Конфігурація та константи
- Розміри вікна та області перегляду
- Шляхи до файлів (модель, іконка)
- Мітки емоцій
- Параметри згладжування та детекції

### `utils.py`
**Призначення:** Допоміжні функції
- `load_font()` - завантаження шрифтів
- `cv2_imread_unicode()` - читання зображень з Unicode підтримкою
- `cv2_imwrite_unicode()` - збереження зображень з Unicode підтримкою
- `preprocess_face()` - попередня обробка обличчя для моделі

### `emotion_model.py`
**Призначення:** Робота з моделлю розпізнавання емоцій
- `load_model()` - завантаження моделі Keras
- `predict_emotion()` - передбачення емоцій на обличчі

### `face_detection.py`
**Призначення:** Детекція облич на зображеннях
- `detect_faces()` - виявлення облич на зображенні
- `get_largest_face()` - отримання найбільшого обличчя

### `video_processing.py`
**Призначення:** Обробка відео та робота з камерою
- `refresh_cameras()` - отримання списку доступних камер
- `open_camera()` - відкриття камери
- `open_video()` - відкриття відео файлу
- `read_frame_camera()` - читання кадру з камери
- `read_frame_video_realtime()` - читання кадру з відео з урахуванням реального часу

### `image_processing.py`
**Призначення:** Обробка та відображення зображень
- `fit_to_view()` - зменшення зображення до розміру вікна
- `compose_on_white()` - розміщення зображення на фоні
- `create_blank_image()` - створення порожнього зображення
- `overlay_annotation()` - додавання анотацій (рамка, текст)
- `convert_frame_to_tk_image()` - конвертація кадру для Tkinter

### `batch_processing.py`
**Призначення:** Пакетна обробка зображень
- `find_image_files()` - пошук файлів зображень у папці
- `analyze_image()` - аналіз одного зображення
- `create_progress_window()` - створення вікна прогресу
- `process_batch_worker()` - робоча функція для обробки пакету

### `main.py`
**Призначення:** Головний файл додатку
- Клас `EmotionApp` - головний клас додатку
- Створення UI
- Управління станом додатку
- Координація роботи всіх модулів

## Переваги такої структури

1. **Модульність** - кожен модуль відповідає за свою область
2. **Читабельність** - легше знайти потрібний код
3. **Підтримка** - простіше вносити зміни та виправляти помилки
4. **Тестування** - можна тестувати модулі окремо
5. **Повторне використання** - функції можна використовувати в інших проектах
